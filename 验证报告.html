🔒安全与健壮性重点

项验证方式结果
只读不改（产品承诺）扫描前后对目标项目全部文件做 md5 哈希比对前后一致，未改任何文件
无 LLM 也可用炸场主线（扫盘/画像/复用/提醒）不依赖 LLM纯本地、秒出
异常不崩溃无效目录 / 缺参数 / 非业务项目三类边界均优雅返回
桌宠掉线不影响评价pet_bridge 推送失败静默忽略评价照常跑
凭证不入库LLM key 走环境变量，grep 全仓库无硬编码 key

↻复现方法

在 hackathon/agent/ 目录：

步骤命令
起评价服务export OPENAI_API_KEY=… OPENAI_BASE_URL=https://api.openai-next.com/v1 CODA_LLM_MODEL=gpt-5 && python3 server.py 8848
CLI 一条龙python3 scan.py samples/clinic-demo
只本地（不花 token）python3 scan.py samples/clinic-demo --no-llm
桌宠工作台右键小哒 → 「打开小哒工作台」→ 📁 选文件夹
测试素材：samples/clinic-demo（项目 A，埋红线，入库）+ samples/clinic-new（项目 B，缺轮子，演复用提醒）。
