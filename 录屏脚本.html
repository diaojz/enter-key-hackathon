90 秒录屏脚本

主轴：桌面小哒 · 金样本：医疗 · 动线：常驻桌面 → 工作台 → 选文件夹扫描 → 三块炸场 → 桌宠变脸 → 跨项目复用

🎯一句话定调（录屏开头字幕 / 路演第一句）

「小哒 Coda，住在你桌面上、懂你行业的 Review 助手。」

「它只读不改——扫一眼你的代码，就猜出你是哪行的，用你那行的话告诉你哪儿有问题、哪些能复用。」

🎬分镜（6 镜头 · 总计 ≤90 秒）

1
小哒登场 · 常驻桌面
0:00–0:10
画面
桌面右下角小哒待机/打盹，鼠标晃一下，它有反应。
操作
右键小哒 → 弹出菜单 → 点「打开小哒工作台」。
旁白
「这是小哒，它一直蹲在我桌面上。点开它的工作台——」
2
选文件夹 · 一键扫描
0:10–0:22
画面
工作台窗口打开（暖米风，三块空卡）。
操作
点「📁 选文件夹」→ 原生对话框 → 选一个医疗项目 → 自动开扫。
画面
同时桌宠切到思考脸（扫盘中）→ 干活脸（评价中）。
旁白
「我授权一个本地项目——注意，它只读不改。小哒开始扫，桌面上它也跟着动起来了。」
3
炸场点 · 我猜你是做医疗的
0:22–0:38
画面
①卡刷出「我猜你是做【医疗】的　置信度 88%」+ 橙色证据词 chips（patient×8、就诊×6…）。
画面
下方「📖 现在在干什么」：用医疗的话讲项目阶段（门诊试运行、前端是叫号屏）。
旁白
「我没告诉它我是哪行的——它扫一眼代码自己猜出来了：医疗，还说出凭据。这一步是全场记忆点。」
4
行话翻译 + 打分（评价核心）
0:38–0:55
画面
③卡：各文件分数徽章（红/橙）+ 展开问题：把「patient 明文存 localStorage」翻成「患者隐私字段裸奔，违反医疗红线」+ 改法建议。
画面
桌宠切到慌张脸（发现红线）。
旁白
「它不说技术黑话，用你这行的话讲问题、给改法——但只给建议，不动你一行代码。踩到红线，小哒自己就急了。」
5
跨项目复用 · 你做过类似的
0:55–1:12
操作
再选第二个医疗项目（项目 B）扫描。
画面
②卡顶部弹橙色💡复用提醒：「你做过类似的医疗项目，有 5 个现成轮子，直接用」——列出患者ID校验 / 就诊状态机，带代码片段+复制。
旁白
「换个新项目，小哒记得你上个项目的积累——『这俩轮子你写过，直接用』。越用越懂你。」
6
人设跟随 · 越用越懂你（收尾）
1:12–1:28
画面
顶部深蓝🧠人设条：「你主要在做【医疗】（2 个项目），越用我越懂你」+ 能力库。
旁白
「扫得越多，它越懂你是谁、擅长什么。这就是小哒——一个常驻桌面、越用越懂你行业的 Review 助手。」
收尾
小哒回到待机，打个招呼 / 字幕「Enter 一下，你的 Agent 就懂你了」。
总时长目标 ≈ 88 秒（留 2 秒缓冲）

✅录前准备清单（关键，照此复位）

录屏前在 hackathon/agent/ 跑一遍复位 + 起服务：
  • 起评价服务：export OPENAI_API_KEY=… OPENAI_BASE_URL=https://api.openai-next.com/v1 CODA_LLM_MODEL=gpt-5 && python3 server.py 8848
  • 桌宠在跑：cd coda-desktop && npm start
  • 清复用库 + 人设（保证第5镜头「项目B」是真·第一次命中，第6镜头人设从 0 累积）：
    python3 -c "from reuse_store import clear_library; clear_library(); from persona import reset; reset()"
  • 预热项目 A：先在工作台扫一次 samples/clinic-demo（让轮子入库），再开始录——录的时候第一个扫 A、第二个扫 samples/clinic-new，复用提醒才会弹。
录屏顺序要点：镜头 2 扫的是项目 A（clinic-demo），镜头 5 扫的是项目 B（clinic-new）。A 必须先入过库，B 才会触发💡复用提醒。若想现场全新演，把"预热"那步也录进镜头 2，镜头 5 自然命中。

🛟兜底与避坑

网络/LLM
评价翻译那步走 openai-next。怕现场抽风：①卡画像/复用/提醒/人设全是纯本地，不依赖 LLM，照常炸场；只有③卡的行话翻译需要 LLM。可提前把③卡结果录进备份录屏。
桌宠竞态
启动日志偶有 Render frame was disposed，是原工程竞态，不影响表情和功能，无视。
key 安全
录屏别把终端里的 API key 录进去；演示用环境变量、画面只露工作台。
必备备份
完整跑通后留一条无瑕疵备份录屏，现场网络异常直接切播放。
