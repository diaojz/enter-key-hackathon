📦两个产品怎么选

🔎 Search API

旗舰级 · 轻量、智能、单接口
  • 只有一个接口，上手最快
  • 固定参数：query · sorting · filter
  • 定位：给应用提供高质量检索能力
  • 也是 Agent 背后的检索引擎
轻量即调即用做检索/RAG 首选

🤖 Agent API

复杂任务 · 云端自动化 · 极简调用
  • 核心是符合 opencli 规范的 responses
  • 为应用构建持续会话流，抛问题它云端自动干完
  • 能力：Deep / Advanced / Academic Research、Code 等
  • 参数多，但有极简默认；用 Preset 快速起步
多步推理会话连续性记忆/Skill

⚙Agent API 调用要点

  • Request Composer：控制台里的 API 体验台，先在这调参数、点 Run 看效果，再落到代码。
  • Preset 是关键：默认 Deep Research。多数场景只需填一个 query + 点 Run，资料收集与分析全在云端自动完成。
  • 示例 query：「帮我分析微软最近一年市场业绩表现、强劲部分、创新点、与竞品的突出差异」→ 直接 Run 即出调研报告。
  • 高阶能力：会话连续性、关联存储、记忆、Skill 都可叠加，针对自己场景做 customization，开放度高。
  • 两种调试路径：① 代码按文档直调；② Playground / Request Composer 手动调参体验。
给 Coding Agent 方向的提示：Agent API 自带「持续会话流 + Code 能力 + 记忆/Skill」，正好是 Coding Agent 的地基——可以省去自己搭 Agent 编排框架的力气，把创新和完成度押在产品层。
