🧭底层原则（创业经验）

反脆弱：不在 AI 必经之路上做创新——不卷提示词这类（模型一更新就白做）。要做 利用 AI、且 AI 越强我们越强 的层面。
创新自检两条：① 通用产品没做的功能你做了 = 创新；② 通用产品有的基础你做得更强（垂域 > 通用）= 有意义。
目标：奔一等奖（综合）· 紧抓最高权重「创新性」· 聚焦解一个问题 · 路演要有真能演示的成品 + 录屏兜底。

🩹起点：小白用户的两大痛点

痛点 1 · 单工具内项目堆叠混乱

用同一个工具（如 Claude）做了一堆项目，散乱堆着，非专业整理不出来。诉求：自动归类 + 整理 + 可视化，不越写越乱。进阶：想做 A 自动定位到 A 接着做，聊到 C 自动切回 C 的上下文。

痛点 2 · 多工具分散，无法整合

薅羊毛/探索，同时用 Codex、Claude、Trae……同一项目多工具试。诉求：把各工具的成果搂到一块儿，不用 A 干完再去 B 复述一遍。跨平台整合是真需求，也是巨头盲区。

🔑关键转向：重新定义用户

不是"小白"，而是——
行业专家 + Coding 新手 · 想成为「Coding 驾驭者 / 转型者」

能在此刻接触 AI Coding 的人，大概率是某行业的资深专家（做管道的、做某垂直业务的），只是对 Coding 不懂。他在自己领域有完整方法论：工作路径、好坏标准、行业历史、人才标准——不该被当小白。

核心机制 = 行业类比：用户先说行业背景 → 产品用他熟悉的领域语言打比方解释 Coding（"这个在你的领域相当于 X"），把 Coding 逻辑 ↔ 专业领域逻辑匹配起来。

📈三层价值上升（商业闭环）

技术层理解 Coding 本身
逻辑层用专业类比理解 Coding 逻辑（语言的可视化 / 说行话，不只图形）
个人/客户层能用自己行业的语言，向领导/客户/下属描绘并解释
→ 商业化接近闭环。这类用户多是行业先驱，背负"教团队用"的任务 → 教会 1 个能裂变教会一群（自带传播），受众更广、赛道更长。

🔀两条候选主线（午饭后二选一 / 融合）

线 A之前收敛的

跨工具项目可视化整合器

自动归类 + 识别 + 连接 + 可视化散乱的 AI 项目；跨工具跨平台聚合视图；移动端可看。逻辑层可视化（能做）→ 技术层可视化（更厉害）。

线 B本轮新增

行业专家的 Coding 转型助手

行业语言类比，把 Coding 翻译成你的领域话；伴随用户从小白成长到专业；帮你向领导/客户解释。

★ 队长情绪上更被这条打动
共同商业锚点：把 Demo 往前推一步到「准生产级」。B 端用户专业挑剔、有显性对手（"我为什么不用豆包？"）；推到准生产 → 才敢给 B 端看 → 才能收反馈 → 正循环。本次先只解决可视化这一个问题。

🏰为什么巨头不做（护城河）

  • 跨工具整合是巨头盲区：Cursor / Trae / Codex / Claude 各自是孤岛，没动力帮你整合竞品——"不方便做"，不是不能做。
  • 本机 / 第三方工具反而能干净地做这件事。
  • 垂域 Agent 必须比通用 Agent 强几个百分点，否则没意义。
