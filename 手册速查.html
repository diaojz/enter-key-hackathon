💡现场打法（实操建议）

  • 核心流程跑通 > 功能堆砌 —— 完成度评分的命门，砍掉一切非核心功能。
  • 赛题一公布立刻锁赛道，全程对着赛道描述做，定期回看别跑偏。
  • 技术深度要露出来：让评委看到清晰的 Agent 设计，不要只是一层 API 调用。
  • 录屏 + GitHub 别拖到最后，提前 1 小时开始整理提交物。
  • 路演前至少演练 2 次，死卡 4 分钟，2 分钟问答想好高频问题。
  • 讲清楚「谁会用、为什么用」——商业潜力维度往往拉开差距。
