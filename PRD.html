与 v2.0 的关键差异

v2.0 用「成长漏斗（入门→进阶→裂变）」叙事，是用户旅程视角，导致「产品到底是个什么东西」始终没讲清。v3.0 改为产品中心：先立住产品形态——一个有形象、常驻桌面的助手（参考 clawd-on-desk 类桌面宠物），再讲它的三个能力：看得见 · 懂业务 · 能裂变，每个能力正面解决一个痛点。

!问题陈述：用 AI 写代码的人，卡在三件事

目标用户是「行业专家但不懂代码」的跨界者。他们想用 Coding Agent 解决本行业问题，却被三件事挡住：

  • ① 看不见（挂着不知道进度）　让 Agent 跑任务，干没干完、要不要确认，得一直盯着终端窗口。盯久了、或网络/token 一失败，就在「门口」放弃了。
  • ② 不懂你（只懂技术不懂业务）　通用 Agent 帮你 Review 只懂技术维度（依赖/结构/bug），不懂你的商业逻辑。它说「变量命名不规范」，却不会说「患者隐私字段不该明文存 localStorage」。用了 AI 但感受不到真正懂我。
  • ③ 传不出（经验沉淀不下、传不出去）　调好的 Prompt、行业规则锁在自己会话历史里，换项目重来，同行更用不上——每个人都在重新发明轮子。

👤目标用户 · 贯穿 Demo 的主线人物

核心受众：T 型人才中想向 π（派）型进化的跨界专家。T 型＝某领域深度专家、技术宽度有限；π 型＝在行业深度之外，能借工具完成基础数字化构建。这类人清楚自己时间价值，付费意愿高。

非目标：职业软件工程师（他们用 Cursor/Claude/Copilot，小哒 Coda 不为他们设计）。

🖥产品形态：一个常驻桌面的 AI 助手

小哒 Coda 不是又一个代码生成器，而是一个住在你桌面上的小助手——有形象、有状态、会反馈。它盯着你的 Coding Agent，懂你的行业，还能把你的经验分享出去。

形象设定 · 小哒 Coda
暖橙色萌系小柴犬精灵，大眼睛、肉垫，住在桌面一角。它就是产品的「脸」——状态全靠它的表情/动作传达。详见 产品命名定稿。

产品载体（参考 clawd-on-desk 类桌面宠物）

  • 桌面常驻形象　像素宠物 / 角色形象常驻桌面一角，不抢屏、随时可见。
  • 状态即形象　思考中💭 / 写代码⌨️ / 跑构建🔨 / 干完了✅ / 等你确认🙋——Agent 在干嘛，看一眼形象就知道，不用盯终端。
  • 跨 Agent 监控　同时盯 Claude Code / Codex / Cursor 等多个 Coding Agent 的会话状态。
  • 能力可插拔　业务知识、行业 Review 规则以插件形式装进来——这是「懂业务」和「能裂变」的技术载体。

技术栈方向：Electron 跨平台桌面应用（Win/macOS/Linux）+ 读取各 Coding Agent 的会话/事件 + Claude Code 原生 plugin 机制承载行业能力。

⚡三个能力（每个正面解决一个痛点）

能力一 · 看得见
桌面助手：让 Agent 的状态浮出水面
解决痛点①：挂着不知道进度、门口就放弃
  • 状态实时反馈　形象随 Agent 活动变化：思考 / 写代码 / 干完了 / 等你确认，不用盯终端
  • 权限气泡 · 一键确认　Agent 要权限时桌面弹卡片，一眼看到、一键放行，不卡在终端
  • 游戏化 · 越用越想用　勋章、宠物互动、进度助推，把折腾变成好玩，提打开率
能力二 · 懂业务
业务版 Code Review —— 核心差异
解决痛点②：通用 Agent 只懂技术、不懂你的行业业务
  • 业务维度 Review　不只查技术，还问「候诊逻辑是否考虑复诊优先排队」——技术对、业务也对
  • 反问补齐，绝不重复问　首次像问卷追问你的行业规则，记住后绝不重复问；「我明明说过了怎么又问」是杀死信任最快的方式
  • 知识不内置，跟着你走　不预置过时的行业知识，你才是专家，它只负责记住、复用你的经验——越用越懂你
能力三 · 能裂变
行业经验打包成插件，同行一键装机
解决痛点③：经验沉淀不下、传不出去 · 48h Demo 主线
「我用了个特别牛的 → 发给你 → 你装机直接感受到强」
  • 经验打包成插件　反问补齐的行业 know-how、调好的脚本，一键打包成 Claude Code 插件
  • 同行一键装机　发个链接，对方 /plugin install 装上，他的 Agent 立刻懂这个行业
  • 越分享越强　同领域垂直 Agent 远强于通用工具，强到让人想推荐 → 垂直社区裂变

🔧技术地基：基于 Claude Code 原生 Plugin 机制

重大利好（已实测）：Claude Code 原生 plugin / marketplace 机制完全 GA，本机 v2.1.195 验证可用。「能裂变」的 Demo 无需自建，现场可真的 /plugin install 一键装插件——评委是技术人，看到真命令执行瞬间建立信任，比任何 PPT 动画都有力。

分享 + 一键装机的真实命令（两步闭环）

# 第一步：注册"技能市场"仓库 /plugin marketplace add enter-key/enter-key-marketplace # 第二步：安装插件 /plugin install enter-key-industry@enter-key-marketplace # 验证 /plugin list

插件能打包什么（承载「懂业务」与「能裂变」）

  • Skills（skills/<name>/SKILL.md）　核心：行业理解与业务 Review 能力
  • Slash commands（commands/*.md）　如 /enter-key:行业需求对齐
  • CLAUDE.md　行业知识库注入（医疗/餐饮/教育词汇与业务规则）
  • Agents / Hooks / MCP　专门化子 Agent、自动触发、工具集成（飞书/钉钉）

桌面助手层技术方向：Electron 桌面应用读取各 Coding Agent 的会话/事件流，渲染形象状态与权限气泡。48h 内桌面助手以原型/录屏呈现，插件能力真跑。

⚠风险与应对

最大风险：Plugin 装前装后对比效果不够显著。整个路演最高潮就是那一帧——无插件通用回复 vs 有插件领域专业回复。立刻开始测对比问题组，把插件里的业务知识写得足够具体、有辨识度。
风险应对
桌面助手 48h 做不完仅做原型/录屏呈现形象与状态，不强求完整 Electron 工程；插件能力真跑兜底
install 命令参数不对今晚必须实测；fallback 直接 Git URL 格式
行业语言效果不稳提前测 20+ 组 prompt，路演只用最稳的三个场景
评委问"和 git clone 啥区别"（必问）plugin 自动注册 skills/commands、有版本管理、一行命令 vs 手动配多文件
现场网络故障三级兜底：手机热点 → 演示机提前装好 → 全程录屏
