🎯产品概述

让不懂代码的行业专业人士，用自己行业的大白话说一句诉求，按下回车，自动生成一个真能跑、能上线、能扫码分享的单页小工具。

回车键 · Enter 一下，世界开始运行

问题陈述

行业专家（诊所运营、餐馆老板、培训机构老师）有明确的数字化提效需求，却卡在两道鸿沟：

  • 能力鸿沟：现有工具（v0、Bolt、Lovable）的交互语言是"组件、路由、数据库、API"，行业专家根本看不懂，不知从何描述需求。
  • 交付鸿沟：即便聊出 Demo，成果停在对话框截图或本地运行，无法生成公网链接分享，没形成真正可用的工具。

我们的解法 · 两座桥

  • 行业语言层：系统自动识别行业背景，用该行业原生术语（"患者、排号、预约时段"而非"用户 ID、数据表、接口"）确认需求 —— 消除能力鸿沟。
  • 一键交付层：Agent 生成后自动部署到公网，输出可访问 URL 和二维码，扫码即用 —— 消除交付鸿沟。

📖用户故事

US-01 · 核心
作为社区诊所前台王姐，我想用"患者来了我要排号"这句话就能生成排号工具，以便不找程序员也能让患者扫码取号。
US-02 · 行业确认
作为小餐馆老板老李，我想系统用"桌号、菜单、后厨通知"这些我听得懂的词确认需求，以便我知道 AI 真理解我，而不是让我猜它在做什么。
US-03 · 生成可见
作为培训机构张老师，我想在 AI 生成报名表时看到它一步步在做什么，以便我知道进度、不觉得系统卡死。
US-04 · 扫码交付
作为任一行业专家，我想生成完成立刻拿到二维码，以便当场扫码验证，满意就发给同事或顾客。
US-05 · 不出现技术词
作为不懂编程的行业专家，我想全程看不到"组件、路由、API、数据库"，以便不会因看不懂而放弃。
US-06 · 路演场景
作为台上路演的创业者，我想"说一句话→回车→扫码可用"在 3 分钟内跑通，以便评委直观感受核心价值。

🧩功能需求 P0 = 48h 必须跑通

① 行业意图层

功能点描述优先级
行业背景采集首次进入选择/输入所在行业（医疗/餐饮/教育/零售/其他），加载对应行业词汇包。P0
大白话输入框纯文字输入，无任何技术提示，占主屏焦点；唯一操作是"回车"或大号 Enter 键。P0
行业语言确认
★ 核心差异化		AI 用该行业术语把需求复述成确认卡"我理解你要做的是……包含……"，用户点"对，开始做"或"不对，我改"；全程零技术词。P0
需求修正循环确认卡上可用自然语言补充/修正，系统重新理解再出卡，最多 2 轮。P1
行业模板推荐输入框下展示 3 条示例诉求（灰色提示），降低首次输入门槛。P2

② 生成层

功能点描述优先级
Agent 多步可见进度调用 Agent API 时，将 意图分析→结构方案→HTML 编写→自检 四步用进度条+行业语言说明实时展示（"正在设计排号逻辑"而非 generating component tree）。P0
单页 HTML 生成严格限定"单页 HTML 小工具"（表单/登记/计算器/展示类），不做多页、不引后端数据库；优先复用主办方 Agent API 的 Code 能力。P0
内置自检步骤生成后自动自检"能否渲染、关键字段是否存在"，发现问题自动重试一次。P0
生成预览部署前内嵌渲染预览，用户可直接交互验证。P1
重新生成预览不满意可带补充描述再跑一次。P1

③ 部署层

功能点描述优先级
自动公网部署生成成功后自动推送公网（Vercel 静态或等效），返回唯一 HTTPS URL。P0
部署状态反馈显示"正在上线，大约需要 XX 秒"，成功/失败均明确提示。P0
部署失败降级失败时提供下载 HTML 兜底 + 可操作说明。P1
URL 有效期说明告知链接有效期（Demo 阶段 7 天或演示期内），设定预期。P2

④ 交付层

功能点描述优先级
二维码生成部署成功即时生成 URL 二维码，醒目展示。P0
一键复制链接"复制链接"按钮，适配非扫码分享。P0
移动端交付页适配交付页在手机浏览器正常显示二维码和链接（台上用手机扫）。P0
生成记录（会话内）本次会话保留历史生成列表，可切换查看。P2

🎬核心流程 · 黄金示例：社区诊所排号工具

1
行业背景采集
王姐打开"回车键"→ 选"医疗"→ 系统加载医疗词汇包（患者/挂号/排号/预约时段/科室/医生），进入输入界面。
2
说行话 · 用户输入
王姐打"我想让患者扫码取号，我这边能看到还有多少人在排队" → 唯一提示是"按回车或点 Enter 键"。
3
行业语言确认 · 核心差异化节点
"我理解你要做的是：一个患者排号小工具。患者扫码后填姓名，系统自动分配排号（001、002……）。你作为医护人员，可在同一页面看到当前排队人数和各号信息。这样对吗？"
王姐说"对，但还要让患者填科室" → 点"修改一下"补充"内科/外科/儿科" → 系统更新确认卡 → 点"对，开始做"。
4
生成 · Agent 多步可见
进度条 + 行业语言描述：① 正在理解排号需求（~5s）② 正在设计排号流程和科室选择（~10s）③ 正在写页面代码（~20–40s）④ 正在检查排号和科室是否正常（~10s）。全程无技术词。
5
预览确认
预览中：患者填姓名、选科室、提交后显示"您的号码是 003，前方还有 2 人"；医护端看实时队列。王姐满意 → 点"上线这个工具"。
6
自动部署
自动推送 Vercel，显示"正在上线，大约 15 秒……" → 返回公网 HTTPS URL。
7
交付 · 扫码即用
交付页：大号二维码 + URL + "复制链接"。王姐手机扫码当场可用，截图发到诊所工作群，患者下次扫码取号。

🚧范围边界（48h）

做（In Scope）

  • 行业背景选择（4 预设 + 其他）
  • 大白话输入 + 回车触发
  • 行业语言确认卡（可修正 1–2 次）
  • Agent 多步生成单页 HTML（表单/登记/取号/计算器/展示）
  • 生成进度实时可见（4 步 + 行业语言）
  • 生成结果内嵌预览
  • 自动部署公网（Vercel）
  • 公网 URL + 二维码交付
  • 移动端可扫码访问交付页
  • 录屏兜底（≥3 段，覆盖诊所全流程）

不做（Out of Scope）

  • 多页应用生成（只做单页）
  • 后端数据库持久化（仅 localStorage / webhook）
  • 账号登录注册系统
  • 历史项目跨会话保存
  • 代码编辑器 / 源码展示
  • 开发者 prompt 调试界面
  • AI Code Review
  • 纯聊天 / 问答套壳
  • 多人协作、版本管理
  • 超过 2 轮的需求修正 / 原生 App

⚙️非功能需求

性能

  • 回车 → 内嵌预览：目标 90s 内，上限 120s；生成期间禁止"空白等待 > 5s"。
  • 部署拿到 URL：目标 20s，上限 30s。
  • 回车 → 扫码可用：目标 2min，上限 3min（路演 6min 内必须跑完一次）。

可靠性

  • 自检失败自动重试一次（最多 2 次生成机会）；2 次均败则给"下载 HTML"兜底，不展示错误堆栈。
  • 路演前完成 ≥3 次完整流程录屏，作为现场网络故障备用。
  • 部署失败提供 HTML 下载 + 一句话说明（"发给任意懂网站的朋友即可上线"）。

可用性 · 零技术词红线

  • 任何界面不得出现：组件、路由、API、数据库、HTTP、JSON、CSS、DOM、变量、函数、部署日志。
  • 确认卡措辞经 ≥1 位非技术人员口头验证可理解；输入界面主操作只有"回车"。
  • 错误提示用行业语言描述问题和下一步，不暴露技术错误。

移动端适配

  • 生成的 HTML 工具在手机浏览器（微信内置/Safari/Chrome）可正常使用。
  • 交付页二维码不小于 200×200px，手机正常渲染。

📊成功指标 · 对齐评分五维

维度权重命中方式
创新性最高"行业语言层"是结构性创新——同类工具从未在生成前用行业术语确认需求，非界面微创新。
产品完成度高Demo 四步全可演示；二维码扫码即用是可验证交付态；现场完成端到端。
技术深度中展示 Agent 多步执行（意图→方案→代码→自检）+ 行业 context 注入对生成质量的影响。
商业价值中"卖铲子"：行业用户付费即得工具，可延伸行业 SaaS 订阅；市场=数亿非技术从业者。
Demo 表现高叙事清晰（问题→解法→演示→扫码）；现场扫码验证是强视觉记忆点。
Demo 验收 Go/No-Go：① 一句行话→回车→行业确认→生成→公网 URL→二维码→手机扫码成功，全程无人工干预；② 确认卡和进度零技术词（人工逐条检查）；③ 完整流程 ≤3min（≥3 次测试达标）；④ 诊所场景录屏 ≥3 段就绪。

⚠️风险与应对

风险 1 · AI 生成不稳定（最高概率）：① 自检自动重试（最多 2 次）；② 收窄到最简表单/展示类；③ 为路演备 3 条黄金场景 preset prompt；④ 录屏兜底。
风险 2 · 网络/Vercel 部署失败：① 路演前提前部署好 3 个成品页存 URL 保底；② 本地 ngrok 隧道备选；③ 交付层提供 HTML 下载。
风险 3 · 场景过宽 48h 做不完：MVP 严收为 3 黄金场景（医疗排号/餐饮点单/教育报名），其余复用"通用表单"；生成严限单页；P1/P2 时间不足全砍。
风险 4 · 撞 v0/Bolt/Lovable：叙事点出差异——它们面向"会描述技术需求的人"，我们面向"连技术需求都不会描述的行业专家"；台上现场做一个 v0 接不住的输入（"我要做个排号的东西"）对比展示行业语言确认层。
