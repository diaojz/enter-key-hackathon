2 在 Render 部署后端

后端零三方依赖、纯 Python 标准库，Render 免费层即可。

  1. 打开 dashboard.render.com → 注册/登录（可用 GitHub 账号）
  2. 右上 New + → Blueprint（蓝图，读仓库里的 render.yaml）
  3. 连接 GitHub → 选 diaojz/enter-key-hackathon 仓库 → Render 自动识别 render.yaml，建出名为 coda-agent 的 Web Service
  4. 填环境变量（Environment 标签，最关键的一步）：
    变量名填什么
    OPENAI_API_KEY主办方中转站那把 key（sk-DY7H…，控制台「API Key」标签）
    CODA_ACCESS_TOKEN自己编一串随机字符串，如 coda-2026-xxxx（护栏，防额度被刷）
    OPENAI_BASE_URLhttps://api.openai-next.com/v1（蓝图已预填，确认即可）
    CODA_LLM_MODELgpt-4o-mini 或 gpt-5（蓝图已预填）
    CODA_LLM_BACKENDopenai（蓝图已预填）
  5. 点 Deploy，等 1～2 分钟出绿色 Live
  6. 记下分配的域名，形如 https://coda-agent.onrender.com
验证：浏览器打开 https://你的域名/health，看到 {"ok": true} 即后端上线。
⚠ 免费层会休眠：15 分钟没人访问会睡，下次访问要等约 30 秒冷启动。路演前 5 分钟先打开一次 /health 唤醒它，避免评委点扫盘时卡在冷启动。
🔒 key 暴露提醒：内置 key 托管在公网，已加 CODA_ACCESS_TOKEN 护栏（请求要带 x-coda-token 头）。桌宠会自动带这个头，但 token 值要和下面步骤③填的一致。

4 出 Mac / Windows 安装包

桌宠用 electron-builder。每个包要焊两个 sidecar：cc-connect-clawd（Go，预编译下载）+ coda-agent（Python 评价后端）。在 coda-desktop 目录执行

🍎 Mac 包（Apple Silicon / arm64）

cd /Users/diaoye/Desktop/coda-desktop
npm install                          # 首次需要

# ① 把 Python 后端打成 arm64 二进制（焊进 bin/coda-agent/darwin-arm64/）
bash scripts/build-coda-backend.sh   # 必须 Python 3.12！3.14 会哑火

# ② 只打 arm64 dmg（关键：别用默认 build:mac）
npm run build:mac:arm64
# → dist/Clawd-on-Desk-0.10.0-arm64.dmg
⚠ 为什么不用 npm run build:mac：默认要打 x64+arm64 双架构，会因缺 cc-connect-clawd 的 darwin-x64 预编译版而在 prebuild 校验阶段失败（exit 0 是假象，dmg 没产出）。build:mac:arm64 是新增脚本，只打 arm64、跳过这个校验。要打 Intel Mac 才需补 x64 sidecar。

🪟 Windows 11 包（x64）

# ① 下 Go sidecar 的 windows-x64 预编译版
#    （node 脚本可能超时，失败就 curl 直连 GitHub Release 手动放进
#     bin/cc-connect-clawd/windows-x64/cc-connect-clawd.exe，校验 SHA256）
node scripts/fetch-sidecar-binaries.js --target windows-x64

# ② 组装 Windows 后端 = 嵌入式 Python + 源码（不走 PyInstaller，见下）
#    把 python-3.12.x-embed-amd64.zip 解开 + agent/*.py + samples/
#    一起放进 bin/coda-agent/windows-x64/

# ③ 出 nsis 安装包（mac 上即可，NSIS 不需要 wine）
npm run build:win:x64
# → dist/Clawd-on-Desk-Setup-0.10.0-x64.exe
❗ Windows 后端为什么不用 PyInstaller：PyInstaller 不能交叉编译——在 mac 上打不出 Windows 的 coda-agent.exe。但本后端是纯标准库零依赖，所以根本不需要打包：直接焊「Windows 嵌入式 Python（embeddable zip）+ 平铺源码」，App 在 Windows 下 spawn python.exe server.py 即可。启动逻辑在 src/coda-backend-sidecar.js（spawnSpec() 按平台分流：win32 走 python.exe，其它平台走单一二进制）。嵌入式 Python 的 python312._pth 含 .，保证同目录 import 生效。
验证盲区：mac 上能验证文件焊入（7zz e 安装包.exe '$PLUGINSDIR/app-64.7z' 再 l 看 resources/sidecars/coda-agent/windows-x64/ 有没有 python.exe + server.py），但无法在 mac 上真跑 windows 的 python.exe。Windows 上首次启动后端能否拉起，要装上后实测「选目录扫本机」。
验证：装一次产出的包，工作台→扫盘，Mac/Win 都确认出「医疗 88%」+ AI 逐文件评审（不是 mock 兜底）。Windows 包焊了本地后端，能扫本机目录，不止云端。
