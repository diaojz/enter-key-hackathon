# 小哒 Coda · 代码质量评分维度底稿

> 调研日期 2026-06-28 · 对标 SonarQube / Code Climate(Qlty) / CodeScene / Codacy / CodeFactor + 学术度量学。
> 为「只读静态扫描 + 不跑测试 + 行业合规红线」的产品形态定制。所有公式/阈值经官方文档查证。

## 一、跨工具评分哲学（对评委讲这段）

业内有两种评分哲学：
1. **连续量化派**（SonarQube Maintainability、Code Climate、CodeScene、微软 MI）：把问题折算成「修复成本/健康度」连续打分。
2. **一票否决派**（SonarQube Reliability/Security）：一个 blocker 直接拉到 E 级。

→ **小哒以「连续量化」为主体打分，对「行业合规红线」采用「一票否决」乘法闸门**——复刻工业界对待安全的做法，有理论背书。

## 二、小哒选定 6 维度

| # | 维度 | 权重 | 能否真算 | 对标背书 |
|---|---|---|---|---|
| 1 | 可维护性 Maintainability | 25% | ✅ 微软 MI 公式真算 | Microsoft VS |
| 2 | 复杂度 Complexity | 20% | ✅ AST 真算 | McCabe 1976 / SonarSource |
| 3 | 可读性/规范 Readability | 20% | ✅ 正则+AST | Code Climate 10 checks |
| 4 | 安全性 Security | 20% | ⚠️ 模式扫描（非逻辑漏洞） | SonarQube / OWASP |
| 5 | 重复度 Duplication | 15% | ✅ token 比对 | SonarQube <3% 绿线 |
| 6 | 合规红线 Compliance | 闸门 | ✅ 正则规则库 | 小哒独有 · 一票否决 |

### 砍掉（诚实标注）
- **测试覆盖率**：❌ 必须跑测试+插桩，小哒不跑测试。降级为「测试文件存在性信号」并加免责声明，绝不伪装成真覆盖率。
- **CodeScene 热点**：❌ 需 git 历史分析，不做。
- **SQALE 技术债比率**：⚠️ 依赖「每行 30 分钟」主观系数，新手难懂，不直接展示。

## 三、权威公式（可引用）

**微软 Maintainability Index（0-100，最值钱·逐字核对微软官网）**
```
MI = MAX(0, (171 − 5.2*ln(Halstead Volume) − 0.23*Cyclomatic − 16.2*ln(LOC)) * 100 / 171)
```
颜色：0–9 红 / 10–19 黄 / 20–100 绿。
> 注意：Halstead Volume 用 log2，MI 用 ln，别混。

**圈复杂度（McCabe 1976）** `M = E − N + 2P` ≈ 决策点数 + 1。分级：1–10 低 / 11–20 中 / 21–50 高 / >50 极高。

**认知复杂度（SonarSource 2018）**：嵌套加重、线性序列不加分、switch 整体只 +1。

**重复行密度** `duplicated_lines / LOC × 100`，SonarQube 绿线 <3%。

## 四、跑分公式骨架

```
小哒总分 = Σ(维度得分 × 权重) × 合规闸门系数

合规闸门（一票否决式·对标 SonarQube 安全 Rating）：
  - 无红线违规       → 系数 1.0
  - 触发警告级红线   → 系数 0.85
  - 触发阻断级红线   → 总分上限锁定 ≤ 59（合规不过则不及格）
```

扣分制（非加分制，对标 SQALE「默认满分、有问题才扣」）：
- 可维护性：直接用微软 MI（已是 0-100）
- 复杂度：基准 100；函数认知复杂度 >15 扣 5，圈复杂度 >10 扣 3
- 可读性：命名不规范 −1/处、关键函数缺注释 −2、方法 >25 行 −2、嵌套 >4 层 −3
- 重复度：100 − 重复行密度% × 系数
- 安全性：高危模式 −15、中危 −5、低危 −1

## 五、「即使算不准也要显得有依据」呈现策略

**核心原则：永远标注方法论出处，把「我们的实现」和「业界标准」绑定。**

| 维度 | UI 标注话术 |
|---|---|
| 可维护性 | 「参考 Microsoft VS Maintainability Index（0-100）」 |
| 复杂度 | 「McCabe 圈复杂度 + SonarSource 认知复杂度双指标」 |
| 重复度 | 「对标 SonarQube 重复行密度（绿线 <3%）」 |
| 安全性 | 「基于 OWASP 模式 + SonarQube 分级思路，静态模式扫描」 |
| 测试 | 「⚠️ 静态扫描不执行测试，此为测试文件信号，非运行时覆盖率」 |
| 合规红线 | 「小哒独有 · 行业合规规则库 · SonarQube 式一票否决」 |

三条技巧：① 每个分数挂方法论 tooltip；② 雷达图 + 🟢🟡🔴 三色分级（CodeScene 9+/4-8.9/1-3.9）；③ 算不准的维度展示「方法论+占位说明」而非假数据——诚实反而显严谨。
