🎯一句话定位

小哒 Coda 是一个常驻桌面、懂你行业的 Review 助手——授权后它扫一遍你的本地项目，反推出你是哪行的，用「你那行的话」把代码问题讲给你听、打分给建议（只读不改）；你越用它越懂你，还能把你攒下的轮子在新项目里直接复用。

回车键 · Enter 一下，世界开始运行

  • 目标用户：用 AI 写代码、但跨项目/跨阶段的代码越攒越乱的行业从业者（既有刚入行的，也有深耕某领域的专业人士）。他们手里有外部工具，缺的是一个「懂业务、拿着鞭子帮我管代码」的助手。
  • 不做：又一个代码生成器（类 v0/bolt）、又一个只懂技术维度的 AI Code Review、纯聊天套壳。它不是代码生成器，是小助手。
  • 核心价值：把跨项目、跨阶段的烂代码，用你那行的语言翻译成你看得懂的问题 + 改法；并让你在同领域的积累跨项目复用，不再重复造轮子。
v4 相对 v3.0 的关键调整：① Demo 高潮由「插件打包→同行装机→装前装后对比」改为「扫存量项目→反推行业画像→行业翻译报告」；② 「懂业务」由纸面/PPT 升为现场真做（半干活：翻译+打分+改法建议，只读不改）；③ 「能裂变（给同行分享插件）」改为「能复用（同一人跨项目复用轮子）」，做医疗最小切片；④ 实时监听收窄到自己的触发场景，降为锦上添花。

🎬Demo 主线（路演就演这条 · 全真）

第一幕 · 主角 · 炸场（扫盘 → 画像 → 行业翻译）

1
授权 · 用户给一个本地项目目录
现场 open 一个医疗类项目（项目 A）。强调只读不可改——系统承诺只看、不删不改任何代码。
2
扫盘 · 关键词拆解
遍历目录，扫变量名 / 文件名 / 注释 / 字符串里的中英文行业词。
3
炸场点 · 反推职业画像
「我猜你是做医疗的」+ 证据（命中了 patient / 就诊 / 药品 / 禁忌症 等词）。评委一眼看懂「看得见 + 懂业务」。
4
行业语言映射报告 + 打分
把 3~5 个代码问题翻译成「医疗行话」，每条带红线等级 + 评分。
5
每条问题挂「建议怎么改」
给文字改法（不自动改）。这就是「半干活」——显得能干活，又不碰用户代码，安全叙事完整。
6
画像卡片 · 可手改
画像可编辑，改完下次扫盘按新画像走——演「越用越懂你」的最小闭环。

行业语言映射示例（报告长这样）

订单状态机缺一步校验→病历流转少了「医嘱复核」，可能漏掉禁忌症
患者字段明文存 localStorage→患者隐私字段裸奔，违反医疗数据红线
时段并发未加锁→同一就诊时段会被重复挂号，候诊顺序会乱

第二幕 · 升华 · 克制（跨项目复用 · 医疗最小切片真做）

7
授权项目 B（另一个医疗新项目）
扫盘命中同行业 → 小哒「你做过类似的项目」。
8
提醒「有现成轮子」 → 点击给出代码片段
行业能力库里有从项目 A 真实抽出的 1~2 个轮子（如患者 ID 校验 / 就诊状态机），当场给出可复用片段。证明「越用越懂你 + 跨项目复用」。
录屏兜底：扫盘 / 画像 / 翻译报告全流程提前录一条完美录屏，现场网络或 Agent 输出异常时切录屏。第二幕需预备两个医疗项目素材（项目 A 已扫、轮子已入库；项目 B 当场扫）——抽取与匹配是当场真跑，素材是合理的 demo 准备。

▢48h 取舍 · 真做 vs 演钩子

核心取舍：火力压在第一幕（扫盘→画像→翻译报告+打分建议）——这是当场能演真的、且最不依赖 hook 稳定性的主线。第二幕复用收窄到医疗 + 1~2 个轮子的最小可信切片（真扫、真抽、真匹配、真给片段），不做通用抽象引擎。实时监听降为锦上添花。

✓ 真做（当场演真的）

  • 扫盘引擎：遍历目录 + 中英文关键词库 + 反推画像
  • Agent 行业翻译 + 打分 + 改法建议（只读不改）
  • 画像卡可手改，改完下次生效
  • 医疗行业能力库 + 跨项目复用 1~2 个轮子

✕ 明确不做

  • 自动改用户代码 / 自动重构（与「只读不改」冲突）
  • 通用跨项目抽象复用引擎（生产级，做不出真的）
  • 全量监听任何通知（收窄到自己的触发场景）
  • 多智能体 / 多用户协同 / 给同行分享（二期）

🧱技术架构（最小可跑）

🔍
扫盘引擎
遍历指定本地目录 + 中英文行业关键词匹配，反推 {行业, 证据词, 文件清单}。本地纯遍历，只读。
🤖
Agent 层
意图识别 + 行业映射 + 改法建议，导出两个 API：/profile 反推画像、/review 出翻译问题+打分+改法。
🖥
前端 / 桌宠
暖米编辑风：桌宠入口 + 画像卡可编辑 + 报告页 + 复用提醒弹窗。对齐既有页配色。

Agent 两个 API（给 Agent 同学的接口契约）

  • POST /profile　入参：扫盘结果；出参：行业画像 + 可编辑字段
  • POST /review　入参：画像上下文 + 单个文件/片段；出参：[{问题, 行业语言翻译, 红线等级, 改法建议}]

详细入参/出参/医疗示例见 Agent 接口契约。

⚖对着五维权重逐条打

维度权重怎么命中
创新性①最高「扫存量代码→反推行业画像→用行业语言翻译问题」是通用 Review 工具没做的组合——它们只懂技术维度，不懂你的业务红线。
产品完成度②次高主线（扫盘→画像→翻译报告）48h 能真跑通，评委随便给个项目目录都能扫，不是固定路径表演。
技术深度③清晰 Agent 链路：扫盘反推 → 意图识别 → 行业映射 → 打分改法；两个 API 边界清晰。
商业价值④卖铲子：服务跨项目/跨阶段代码越攒越乱的行业从业者，付费意愿高；复用线指向「越用越懂你」的留存。
Demo 表现⑤放大器「扫一眼就猜出你是哪行的 + 用你的行话讲问题」当场可体验，反差感最炸。
避坑：别让 demo 退回「自然语言生成网页」（撞 v0 且打不过），也别让它停在「只报信不干活」。开场就把「扫存量代码→反推行业」打出来，把行业翻译层做成全场记忆点。
