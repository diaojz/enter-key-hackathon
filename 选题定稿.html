🎯一句话定位

让不懂代码的行业专业人士，用自己行业的大白话说一句诉求 → 一回车 → 自动生成一个真能跑、能上线、能扫码分享的小工具。

回车键 · Enter 一下，世界开始运行

  • 目标用户：行业专家 + Coding 新手（背着 AI 指标、要给团队做提效工具的非程序员）
  • 不做：专业开发者的 IDE 增强、不做又一个 AI Code Review、不做纯聊天套壳
  • 商业逻辑：卖铲子（不抢大模型地盘）+ 嵌入式（做工作流的一块）+ 直接交付成果

🎬Demo 主线（路演就演这条）

1
说行话 · 用户输入行业大白话
例：「我是做社区诊所的，想要一个让患者扫码登记预约、按时段排号的小页面」——全程不提一个技术词。
2
回车 · AI 用你的行业语言确认理解
产品反问/确认用的是医疗行业的话（"预约时段、排号、患者信息"），不是"数据库表、API"。这是差异化护城河。
3
生成 · Agent 自动写出可运行代码
调用 Agent API 的 Code 能力，多步：意图→方案→代码→自检。屏幕上能看到 Agent 在干活（露技术深度）。
4
交付 · 直接产出公网链接 + 二维码
不是给代码、不是给 Demo，是一个当场扫码就能用的成品。评委手机扫一下就能操作——全场最炸的瞬间。
录屏兜底：AI 有不确定性，同一句话现场可能出不来。提前录一条完美的全流程录屏，现场网络/出图异常时切录屏，评委按实际展示评分。

⚖对着五维权重逐条打（这就是为什么选它）

维度权重怎么命中
创新性①最高「行业语言确认 + 直接出可上线成品」是 v0/豆包没做齐的组合——它们要么给代码（小白看不懂），要么给 Demo（上不了线）。
产品完成度②次高场景收窄到「生成单页可用小工具」，48h 能真跑通，不是演固定路径。
技术深度③清晰 Agent 链路：意图理解→行业类比→代码生成→自动部署，正好吃 Agent API 的 Code 能力。
商业价值④卖铲子：服务背 AI 指标、要给团队做工具的行业人；嵌入式留扩展。
Demo 表现⑤放大器「一句话→回车→扫码就能用」30 秒讲清、当场可体验，最炸。

🔍竞品验证 基于已知信息，联网复核待补

  • v0 / bolt.new / Lovable / Replit Agent：都能"自然语言→生成可运行前端/全栈+部署链接"，是最接近的赛道。但它们面向会描述技术需求的人，交互语言是产品/技术语，不做行业语言类比，也不假设用户是某行业专家。
  • Claude Artifacts / 豆包 / 扣子(Coze)：能生成小工具/卡片，但多停在会话内 Demo，缺"一键出公网可访问成品 + 嵌入用户工作流"。
  • Cursor / Trae / 通义灵码：面向专业开发者的 IDE 增强，跟我们用户群正交，不撞。
结论：三点齐全（① 行业大白话输入 ② 自动部署出公网链接 ③ 面向非程序员行业专家）的产品暂未见到，差异化空位成立。最该强调的差异点 = 「行业语言层」——这让我们是"垂域 > 通用"，而不是"又一个 v0 套壳"。务必让评委记住这一点。
避坑：别把 demo 做成"自然语言生成网页"——那跟 v0 撞且打不过。开场第一句就强调"用户说的是行业黑话，不是需求文档"，把行业翻译层做成全场记忆点。

  • MVP 把生成目标收窄到"单页 HTML 小工具"（表单/登记/计算器/展示类），最容易 48h 跑通且能真部署。
